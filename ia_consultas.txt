PROMPT SECUNDARIO 2: Validaciones Robustas y Seguridad
================================================================================

Implementar validaciones y seguridad para formularios:

1. Validación de Email:
   - Regex robusta que acepte formatos comunes: user@domain.ext, user+tag@domain.co.uk
   - Función validarEmail(email) que retorne boolean

2. Validación de Contraseña:
   - Mínimo 8 caracteres
   - Función validarPassword(password) que retorne {valido, error}
   - Función validarCoincidenciaPassword(pass1, pass2)

3. Validación General:
   - Función validarCampos(campos) que retorne {valido, errores}
   - No permitir campos vacíos

4. Seguridad contra XSS:
   - Función escapeHtml(str) que convierta <, >, &, ", ' a entidades HTML
   - Usar siempre al insertar contenido de usuario en DOM

5. Hashing de Contraseña (Demo):
   - Función hashPassword(password) usando crypto.subtle.digest('SHA-256')
   - Comentar que es SOLO para demostración educativa
   - En producción usar bcrypt en servidor

6. Gestión de Errores:
   - Mostrar errores inline bajo cada campo
   - Clase .error en elementos <span>
   - Función mostrarError(fieldId, errorId, mensaje)
   - Función limpiarError(fieldId, errorId)

RECOMENDACIONES DE SEGURIDAD DOCUMENTADAS
================================================================================

1. CLIENT-SIDE (implementado en este proyecto):
   - Validación de entrada
   - Escape HTML contra XSS
   - Hashing simulado (SHA-256)

2. SERVER-SIDE (implementar en producción):
   - ¡NUNCA confiar en validaciones de cliente!
   - Usar bcrypt o Argon2 para hashing (NO SHA-256)
   - HTTPS obligatorio
   - JWT con expiración para sesiones
   - SQL injection prevention (queries parametrizadas)
   - Rate limiting en endpoints de auth
   - CORS bien configurado
   - CSP headers
   - Logout seguro (eliminar todas las sesiones)

3. ALMACENAMIENTO (producción):
   - NO guardar contraseñas en localStorage
   - JWT en httpOnly cookie
   - Sesiones en base de datos del servidor
   - Datos de usuario cifrados en tránsito

4. COMENTARIOS EN CÓDIGO:
   - Cada función de seguridad tiene comentarios explicativos
   - Señalar dónde se necesita implementación en servidor
